# Notebooks — qqc-004-verification-equivalence

This paper package does not currently include executable notebooks.

The trust-layer examples and diagnostics reported in the paper are generated by
the shipped Python verifier in `artifacts/code/verify_canonical_equivalence.py`,
not by a notebook workflow.

If future notebooks are added, they should remain supporting material rather
than primary evidence. Suitable notebook topics include:

- visualizing the canonical hemisphere and equatorial tie-break convention,
- exploring how trace overlap and process fidelity behave under small
  perturbations of a canonical `u1q` payload, and
- demonstrating interoperability between quaternionic and matrix-based
  verification pipelines.

Any future notebook should state clearly whether it is pedagogical, regression
support, or prototype tooling for a downstream compiler or API repository.
