Quaternionic Frame Residuals and Coherence Metrics for Lunar and Cislunar Navigation Software Confidence Scoring

Technical first draft of WaveEngine Nav as a diagnostic confidence layer for autonomous lunar and cislunar navigation pipelines.

John G. Van Geem
RQM Technologies
ORCID: 0009-0002-4003-8452

Abstract

Autonomous lunar and cislunar missions must sustain navigation performance under sparse measurements, delayed communications, GPS-denied operations, intermittent relay geometry, clock drift, and degraded optical or terrain-relative sensing. This paper defines a quaternionic confidence framework for evaluating the internal consistency of position, velocity, attitude, timing, and measurement geometry in such regimes. The proposed WaveEngine Nav layer computes normalized measurement innovations, quaternionic frame residuals, observability conditioning indicators, clock-drift consistency terms, and dynamics consistency terms, then combines them into an interpretable PNT confidence score. The framework is intended to augment, not replace, conventional navigation filters and orbit-determination workflows. Its contribution is a structured diagnostic method for identifying under-observed, frame-inconsistent, timing-degraded, or dynamically implausible navigation states before those states evolve into an obvious loss-of-confidence event.

1. Introduction

WaveEngine Nav is scoped as a navigation confidence layer for lunar and cislunar missions where sparse observability, delayed communication, and variable relay geometry can mask pending state-quality degradation. The framework is designed to run beside existing estimator pipelines rather than replacing them.

2. Scope, limitations, and non-claims

This draft does not claim a new orbit-determination method, does not replace EKF/UKF/factor-graph workflows, does not claim universal quaternion superiority, and does not present flight-qualified implementation results.

3. Navigation state and measurement model

State form: x_k = [r_k, v_k, q_k, b_k, dot{b}_k]^T. Measurement form: z_k = h(x_k) + eta_k. Representative predictions include range and Doppler terms augmented by clock bias and drift.

4. Quaternionic frame residuals

Line-of-sight mapping uses l_body = q^{-1} l_inertial q. With observation-derived quaternion q_obs, mismatch is q_delta = q_obs q_est^{-1} and theta_delta = 2 arccos(|w_delta|), interpreted as a geodesic mismatch on S^3/SU(2).

5. Normalized innovation and residual structure

Innovation is e_k = z_k - h(x_k) and covariance-normalized score is d_k^2 = e_k^T S_k^{-1} e_k. Residual magnitude is treated as one diagnostic component rather than a complete confidence indicator.

6. Observability and geometry conditioning

With H_k = partial h / partial x and O_k = H_k^T R_k^{-1} H_k, condition-number and singular-value trends indicate geometry weakness, including sparse cislunar tracking and intermittent lunar relay cases.

7. Coherence score

Component scores C_residual, C_frame, C_clock, C_observability, and C_dynamics combine as C_nav = Σ w_i C_i with mission-configured weights.

8. Candidate algorithm

The candidate algorithm ingests filter state and measurements, computes normalized innovations and frame residuals, evaluates observability and timing consistency, then emits C_nav, degradation mode labels, and a suggested next measurement class.

9. Lunar and cislunar use cases

Conceptual use cases include (A) intermittent lunar relay support for a surface asset, (B) sparse Earth-tracking for cislunar autonomy and scheduling, and (C) low-light terrain-relative or optical navigation near the lunar south pole.

10. Phase I validation path

Proposed simulations include injected clock drift, attitude misalignment, weak relay geometry, Doppler outliers, optical/TRN outliers, and delayed updates. Success criteria are framed as proposed validation targets, not completed results.

11. Conclusion

This draft defines a quaternionic coherence framework to monitor confidence in navigation solutions before overt estimator failure. The layer is intended as a complement to existing flight-dynamics and navigation software stacks.
